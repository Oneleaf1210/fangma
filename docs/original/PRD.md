## 1. 文档概述

### 1.1 产品简介
**放自己一马**（简称**放马**）是一款帮助现代人在快节奏生活中找到暂停时刻的手机应用。通过马匹管理隐喻体系（放马、策马、驭马、抓马）和AI陪伴，为用户提供简单易用、有趣、有惊喜的放松体验，并结合趣味语言学习获得aha moment。

### 1.2 核心概念：马匹管理隐喻体系
- **放马**：暂停休息，给自己放松时刻
- **策马**：回归工作，重新出发的动力
- **驭马**：自我管理，掌控生活节奏
- **抓马**：惊喜时刻，有趣的发现和aha moment

### 1.3 目标平台
- 微信小程序（优先）或 HTML5 移动网页应用
- 优势：无需安装、快速传播、开发成本低、易于迭代

## 2. 项目概述

### 2.1 产品愿景
通过马匹管理隐喻帮助现代人在快节奏生活中找到平衡，通过有趣的AI陪伴和语言学习获得惊喜时刻，实现轻松减压和个人成长。

### 2.2 核心价值
- **简单易用**：极简界面，新用户1分钟内完成首次设置
- **有趣体验**：马匹隐喻、趣味语言学习、惊喜时刻
- **有温度陪伴**：AI情绪支持，提供有温度的对话
- **实用价值**：粤语/英语俚语学习，提升语言能力

### 2.3 成功标准（KPI）
- 用户每日使用时长 > 3分钟
- 用户留存率（7日） > 30%
- 用户满意度评分 > 4.5/5
- 用户aha moment（抓马时刻）触发率 > 20%
- 月活跃用户（MAU）增长率 > 15%

## 3. 目标用户分析

### 3.1 用户画像
#### 主要用户群体
1. **25-40岁上班族**
   - 特征：工作压力大，经常加班，需要情绪支持
   - 需求：工作间隙放松、压力缓解、正能量鼓励
   - 使用场景：办公室、通勤途中、午休时间

2. **大学生群体**
   - 特征：学业繁忙，未来焦虑，需要鼓励和安慰
   - 需求：学习间隙放松、情绪调节、语言学习
   - 使用场景：图书馆、宿舍、自习室

3. **自由职业者**
   - 特征：工作时间不固定，缺乏工作生活边界
   - 需求：节奏调整、时间管理、自我激励
   - 使用场景：居家办公、咖啡馆、共享空间

### 3.2 使用场景
1. **工作间隙休息**：3-5分钟短暂放松，缓解工作压力
2. **情绪低落陪伴**：需要情绪支持和鼓励的时刻
3. **语言学习兴趣**：利用碎片时间学习有趣的语言知识
4. **睡前放松**：睡前短暂放松，改善睡眠质量
5. **创意激发**：需要灵感时的头脑放松

## 4. 功能需求

### 4.1 功能架构（按马匹隐喻组织）
```
放马功能（暂停休息与放松）
├── 暂停提醒
├── AI情绪支持对话
└── 呼吸练习（扩展）

策马功能（回归工作与动力）
├── 回归提醒
└── 动力鼓励

驭马功能（自我管理与控制）
├── 用户设置管理
├── 数据存储与同步
├── 隐私设置
└── 学习记录查看

抓马功能（惊喜时刻与语言学习）
├── 语言学习模块
├── aha moment设计
├── 成就系统（扩展）
└── 心情记录（扩展）

AI核心支持
├── 对话历史记录
└── 多场景prompt支持
```

### 4.2 详细功能说明

#### 4.2.1 放马功能：暂停休息与放松
**核心MVP功能**
- **暂停提醒**
  - 功能描述：自定义提醒用户停下来休息
  - 详细要求：
    - 可设置提醒时间（具体时间点或间隔时间）
    - 可设置提醒频率（每日、工作日、自定义）
    - 提醒方式：通知推送、声音提醒、震动提醒
    - 提醒内容：温馨文案，如"该放马休息一下了~"
    - 支持勿扰模式设置

- **AI情绪支持对话**
  - 功能描述：自然语言对话，提供情绪支持和鼓励
  - 详细要求：
    - 识别用户情绪状态，提供针对性支持
    - 鼓励性话语和正能量回应库
    - 简单的认知行为疗法（CBT）技巧引导
    - 支持连续对话，保持上下文连贯性
    - 对话风格：温暖、亲切、支持性

- **呼吸练习**（扩展功能）
  - 功能描述：引导式呼吸放松练习
  - 详细要求：3-5分钟呼吸引导，可视化呼吸节奏

#### 4.2.2 策马功能：回归工作与动力
- **回归提醒**
  - 功能描述：提醒用户结束放松，回归工作/生活
  - 详细要求：
    - 可设置放松时长（默认5-15分钟，可自定义）
    - 结束时提醒文案："该回归做牛马了"或"策马前行"
    - 提供正能量鼓励，帮助用户顺利过渡
    - 支持跳过/延长选项

- **动力鼓励**
  - 功能描述：回归时的正能量话语和鼓励
  - 详细要求：积极心理学语录，目标设定引导

#### 4.2.3 驭马功能：自我管理与控制
- **用户设置管理**
  - 功能描述：用户账户和偏好设置
  - 详细要求：
    - 微信授权登录或手机号快速注册
    - 提醒时间设置界面
    - 放松时长设置
    - 语言学习偏好设置（粤语/英语比例）
    - 通知偏好设置

- **数据存储与同步**
  - 功能描述：用户数据存储和管理
  - 详细要求：
    - 本地存储用户设置和对话历史
    - 云备份和跨设备同步
    - 数据加密存储

- **隐私设置**
  - 功能描述：用户隐私控制
  - 详细要求：
    - 数据收集和使用说明
    - 隐私开关控制
    - 数据删除功能

- **学习记录查看**
  - 功能描述：查看历史学习内容和进度
  - 详细要求：
    - 学习历史时间线
    - 已学俚语列表
    - 学习统计（学习天数、掌握俚语数）

#### 4.2.4 抓马功能：惊喜时刻与语言学习
- **语言学习模块**
  - 功能描述：在放松时刻学习有趣的语言知识
  - 详细要求：
    - **粤语俚语学习**：每日一句常用粤语俚语及解释
      - 例句：实际使用场景
      - 发音指导：拼音或音频
      - 文化背景：俚语来源和含义
    - **英语俚语学习**：每日一句实用英语俚语及使用场景
      - 例句：真实对话场景
      - 使用场景说明
      - 文化差异解释
    - 学习形式：卡片式展示，简洁明了

- **aha moment设计**
  - 功能描述：设计惊喜时刻，让用户在学习中获得快乐和启发
  - 详细要求：
    - 惊喜的语言知识点（非常规但有趣的内容）
    - 有趣的俚语背后的文化故事
    - 学习成就解锁时的庆祝效果
    - 个性化推荐的学习内容

- **成就系统**（扩展功能）
  - 功能描述：完成暂停任务的简单奖励
  - 详细要求：
    - 徽章系统：连续使用、学习进度等
    - 成就解锁动画
    - 分享功能

- **心情记录**（扩展功能）
  - 功能描述：简单记录当前心情和想法
  - 详细要求：表情选择+简短文字，生成心情日历

#### 4.2.5 AI核心支持
- **对话历史记录**
  - 功能描述：保存最近对话和学习记录
  - 详细要求：最近5-10轮对话历史，支持查看

- **多场景prompt支持**
  - 功能描述：为不同功能提供针对性的AI交互
  - 详细要求：基于马匹隐喻的prompt设计

## 5. 非功能需求

### 5.1 性能需求
- 页面加载时间 < 2秒
- AI响应时间 < 3秒（网络良好情况下）
- 界面交互响应时间 < 100ms
- 应用启动时间 < 1.5秒

### 5.2 可用性需求
- 新用户1分钟内能完成首次暂停设置
- 界面操作符合直觉，无需教程
- 错误提示清晰，提供解决方案
- 支持无障碍访问

### 5.3 安全性需求
- 用户数据加密存储
- 传输数据使用HTTPS
- 符合GDPR和隐私政策要求
- 定期安全审计和漏洞修复

### 5.4 可靠性需求
- AI服务可用性 > 99%
- 优雅降级处理：AI服务不可用时提供基础功能
- 错误恢复机制
- 数据备份和恢复

### 5.5 兼容性需求
- 支持微信小程序最新版本
- 或支持主流移动浏览器（Chrome, Safari等）
- 适配主流屏幕尺寸（手机端）
- 支持iOS和Android系统

## 6. 用户体验设计

### 6.1 用户流程（基于马匹隐喻）
1. **驭马**（设置阶段）
   ```
   首次启动 → 品牌介绍 → 马匹隐喻说明 →
   快速设置提醒时间 → 设置放松时长 → 语言学习偏好设置
   ```

2. **放马**（放松阶段）
   ```
   暂停提醒触发 → 进入放马界面 →
   选择活动（AI对话/语言学习）→ 开始放松体验
   ```

3. **抓马**（惊喜时刻）
   ```
   在放松中发现有趣的语言知识点 →
   文化故事讲解 → aha moment触发 → 成就解锁
   ```

4. **策马**（回归阶段）
   ```
   放松结束提醒 → 正能量鼓励 →
   目标设定建议 → 回归工作准备
   ```

5. **驭马**（管理阶段）
   ```
   查看学习记录 → 调整设置 →
   查看成就 → 持续优化体验
   ```

### 6.2 视觉隐喻设计
- **放马界面**
  - 视觉元素：草原、蓝天、白云、放松的小马图标
  - 色彩方案：浅蓝色为主，代表放松和平静
  - 动效：缓慢飘动的云朵，轻松的音乐

- **策马界面**
  - 视觉元素：道路、前进箭头、奔跑的马匹图标
  - 色彩方案：橙色为主，代表动力和活力
  - 动效：前进箭头闪烁，激励性音效

- **驭马界面**
  - 视觉元素：缰绳、设置齿轮、控制面板元素
  - 色彩方案：深蓝色为主，代表控制和稳定
  - 布局：清晰的功能分区，直观的操作

- **抓马界面**
  - 视觉元素：惊喜礼盒、闪光效果、戏剧性视觉元素
  - 色彩方案：金色/黄色为主，代表惊喜和收获
  - 动效：礼盒打开动画，庆祝音效

### 6.3 交互设计原则
1. **极简主义**：减少认知负荷，核心功能突出
2. **一致性**：马匹隐喻贯穿所有界面
3. **反馈及时**：操作有明确视觉或触觉反馈
4. **容错性**：支持撤销操作，错误提示友好
5. **无障碍**：支持屏幕阅读器，色彩对比度达标

## 7. 技术架构

### 7.1 平台选择
**方案一：微信小程序**（推荐）
- 优势：无需下载安装，用户获取成本低，社交分享便利
- 技术栈：微信小程序框架 + 云开发

**方案二：HTML5移动网页应用**
- 优势：跨平台，无需应用商店审核
- 技术栈：Vue.js/React + 响应式设计

### 7.2 技术栈详细说明
#### 前端技术
- **微信小程序方案**：
  - 框架：微信小程序原生框架
  - UI组件：Vant Weapp 或自定义组件
  - 状态管理：小程序原生或MobX-miniprogram
  - 网络请求：wx.request + 拦截器

- **HTML5方案**：
  - 框架：Vue 3 或 React 18
  - UI组件：Vant Mobile 或 Ant Design Mobile
  - 状态管理：Pinia 或 Redux Toolkit
  - 构建工具：Vite 或 Webpack

#### 后端技术
- **语言**：Node.js (Express/NestJS) 或 Python (FastAPI)
- **数据库**：
  - 主数据库：PostgreSQL（用户数据、学习记录）
  - 缓存：Redis（会话缓存、频率限制）
- **API设计**：RESTful API + GraphQL（可选）
- **身份认证**：JWT + OAuth 2.0

#### AI集成
- **AI服务提供商**：
  - 主要：Anthropic Claude API（推荐，成本效益高）
  - 备用：OpenAI GPT API
  - 国内备选：通义千问、文心一言（如需国内服务）

- **prompt工程**：
  - **放马prompt**：放松引导、情绪支持、减压技巧
  - **策马prompt**：动力鼓励、目标设定、正能量语录
  - **驭马prompt**：设置指导、使用帮助、进度反馈
  - **抓马prompt**：语言学习、文化故事、惊喜设计

- **成本控制**：
  - 用户每日使用限额
  - 缓存常用回应
  - 优化prompt长度
  - 监控API使用情况

#### 第三方服务集成
- **推送通知**：
  - 微信小程序：模板消息
  - HTML5：浏览器通知 + 服务端推送
- **数据分析**：
  - 微信小程序：微信数据分析
  - HTML5：Google Analytics + 自定义事件跟踪
- **错误监控**：Sentry
- **云存储**：腾讯云COS 或 AWS S3
- **CDN**：腾讯云CDN 或 Cloudflare

## 9. 风险与假设

### 9.1 技术风险
1. **AI服务成本控制**
   - 风险：API调用成本超出预算
   - 缓解：设置使用限额，优化prompt，缓存常用回应

2. **微信小程序审核限制**
   - 风险：AI对话内容可能触发审核问题
   - 缓解：内容过滤机制，人工审核关键内容

3. **语言学习内容准确性**
   - 风险：俚语解释不准确或文化背景错误
   - 缓解：人工审核内容库，用户反馈机制

### 9.2 内容风险
1. **文化敏感性**
   - 风险：俚语内容可能包含敏感或冒犯性含义
   - 缓解：多语言专家审核，文化背景说明

2. **马匹隐喻的清晰传达**
   - 风险：用户不理解隐喻含义
   - 缓解：清晰的引导说明，直观的视觉设计

### 9.3 市场风险
1. **用户接受度**
   - 风险：用户对语言学习式放松不感兴趣
   - 缓解：MVP验证，用户调研，迭代优化

2. **目标用户对粤语学习的兴趣**
   - 风险：主要用户群体对粤语学习需求不足
   - 缓解：提供英语俚语作为主要选择，粤语为特色

3. **竞争分析**
   - 风险：类似应用竞争
   - 缓解：差异化定位（马匹隐喻+语言学习）

### 9.4 运营风险
1. **内容安全审核**
   - 风险：用户生成内容或AI生成内容安全问题
   - 缓解：内容过滤，人工审核，举报机制

2. **用户隐私保护**
   - 风险：数据泄露或滥用
   - 缓解：加密存储，最小化数据收集，透明隐私政策

3. **aha moment设计难度**
   - 风险：惊喜时刻设计不够有效
   - 缓解：用户测试，数据分析，持续优化

### 9.5 关键假设
1. **用户需求假设**
   - 用户有暂停休息和情绪支持的需求
   - 用户对轻松的语言学习感兴趣
   - 用户能理解并接受马匹管理隐喻

2. **市场假设**
   - 目标用户群体规模足够支持应用发展
   - 微信小程序/H5是合适的发布平台

3. **技术假设**
   - AI API服务稳定可靠
   - 开发团队具备所需技术能力
   - 项目时间估算合理
