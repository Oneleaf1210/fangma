# 放马问卷部署检查清单

## 📋 部署前准备

### 账户和权限
- [ ] **GitHub账户**
  - 已有GitHub账户
  - 可以创建公开仓库
- [ ] **Google账户**
  - 已有Google账户（Gmail）
  - 可以访问Google Sheets和Google Apps Script

### 本地准备
- [ ] 下载问卷文件
- [ ] 安装文本编辑器（用于修改代码）
- [ ] 现代浏览器（Chrome/Firefox/Edge）

## 🚀 第一阶段：GitHub Pages部署

### 1. 创建GitHub仓库
- [ ] 访问 https://github.com/new
- [ ] 仓库名: `fangma-survey-data`
- [ ] 描述: "放马产品市场调研问卷数据收集"
- [ ] 选择 **Public**（公开）
- [ ] 不初始化README（我们已有文件）
- [ ] 点击 **Create repository**

### 2. 上传问卷文件
- [ ] 在仓库页面点击 **Upload files**
- [ ] 拖放或选择问卷文件：
  - `放马产品市场调研问卷_带数据收集.html`
- [ ] 提交信息: "初始提交：放马产品调研问卷"
- [ ] 点击 **Commit changes**

### 3. 启用GitHub Pages
- [ ] 进入仓库 **Settings**
- [ ] 左侧菜单选择 **Pages**
- [ ] Source: 选择 **main** 分支，**/ (root)** 目录
- [ ] 点击 **Save**
- [ ] 等待几分钟，直到显示 "Your site is published at..."
- [ ] 记下访问URL: `https://[用户名].github.io/fangma-survey-data/放马产品市场调研问卷_带数据收集.html`

### 4. 验证部署
- [ ] 访问上面的URL
- [ ] 确认问卷页面正常加载
- [ ] 检查页面样式和功能是否正常
- [ ] 填写测试数据（不要提交）

## 📊 第二阶段：Google Sheets数据收集

### 5. 创建Google表格
- [ ] 访问 [sheet.new](https://sheet.new)
- [ ] 重命名表格为"放马问卷数据"
- [ ] 设置表头（A1-O1）：

```
提交时间 | 年龄 | 职业 | 压力程度 | 概念理解 | 学习兴趣 | 语言偏好 | 功能偏好 | 放松时长 | 使用场景 | 价格接受度 | 付费功能 | 推荐意愿 | 建议 | 联系方式
```

### 6. 创建Google Apps Script
- [ ] 在Google表格中：**扩展程序 → Apps Script**
- [ ] 删除默认代码
- [ ] 复制 `google-apps-script.js` 中的代码
- [ ] 粘贴到编辑器
- [ ] 保存项目（Ctrl+S）
- [ ] 项目名称: `放马问卷数据收集`

### 7. 部署为Web应用
- [ ] 点击 **部署 → 新建部署**
- [ ] 类型选择：**Web 应用**
- [ ] 配置设置：
  - 说明：放马问卷数据收集API
  - 执行身份：**我**（您的Google账号）
  - 访问权限：**任何人**（重要！）
- [ ] 点击 **部署**
- [ ] 授权Google账号（点击"Review Permissions"）
- [ ] 选择您的Google账号
- [ ] 点击 **Allow**
- [ ] **复制Web应用URL**（格式：`https://script.google.com/macros/s/XXXXXXXXXX/exec`）

### 8. 配置问卷文件
- [ ] 下载问卷文件到本地（如果需要）
- [ ] 用文本编辑器打开 `放马产品市场调研问卷_带数据收集.html`
- [ ] 找到第770行附近：
  ```javascript
  const GOOGLE_SCRIPT_URL = 'https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec';
  ```
- [ ] 将 `YOUR_SCRIPT_ID` 替换为实际的Script ID
- [ ] 保存文件

### 9. 更新GitHub仓库
- [ ] 返回GitHub仓库
- [ ] 点击上传文件
- [ ] 上传更新后的问卷文件
- [ ] 提交信息: "更新Google Script URL配置"
- [ ] 提交更改

## 🔧 第四阶段：故障排除和优化

### 14. 常见问题检查
- [ ] Google Script返回403错误？
  - 重新部署，确保访问权限为"任何人"
- [ ] 数据未保存到Google Sheets？
  - 检查浏览器控制台错误
  - 检查Network标签请求状态
- [ ] 表单提交卡住？
  - 检查网络连接
  - 禁用广告拦截插件
  - 刷新页面重试

### 15. 性能优化
- [ ] 问卷页面加载时间 < 3秒
- [ ] Google Script响应时间 < 5秒
- [ ] 移动设备兼容性测试

### 16. 安全性检查
- [ ] 问卷通过HTTPS访问
- [ ] Google Script仅收集必要数据
- [ ] 本地存储数据可清理
